الموقع في حالة وظيفية جيدة بشكل عام. التصميم متسق، البنية المعمارية سليمة، ونظام المصادقة والاشتراكات يعمل. هناك نقاط قوة حقيقية وبعض الثغرات التي تحتاج انتباه.

## الممتاز (يستحق التنويه)

### 1. نظام المصادقة المتين
- Magic Link + OTP عبر الجوال — خيارين للمستخدم
- تطبيع أرقام الهواتف السعودية تلقائياً (05 → +966)
- Cooldown ذكي يمنع الإرسال المتكرر مع عداد مرئي
- حماية Rate Limit مع رسائل واضحة بالعربي
- Auth Callback يعالج Magic Link و OAuth/PKCE بشكل سليم
- التوجيه الآمن (`safeNext`) يمنع Open Redirect

### 2. نظام الاشتراك والتفعيل
- `/api/activate` محمي ومنظم: يتحقق من الكود → يحدّث الكود → يحدّث البروفايل → يكتب Entitlement Cookie
- الباقة السنوية تمنح تلقائياً: `book_access` + `tasbeeh_access`
- الأكواد محمية من الاستخدام المزدوج (`used_by` check)
- Entitlement Token بـ HMAC-SHA256 — آمن وسليم

### 3. تجربة اليوم (DayExperience)
- تدفق مدروس: SilenceGate → VerseBlock → HiddenLayer → BookQuote → ReflectionJournal → AwarenessMeter → ShareCard → ProgressionBadge
- Auto-save للتأملات كل 1.5 ثانية
- مشاركة ذكية (Web Share API مع fallback للـ clipboard)
- Milestone badges في الأيام المحورية (1, 3, 7, 14, 21, 28)

### 4. صفحة البرنامج `/program`
- عرض مرئي واضح: 4 أسابيع، كل يوم بحالة (مكتمل/اليوم/متاح/مغلق)
- Streak محسوب بشكل صحيح
- شريط تقدم بنسبة مئوية
- حماية 401 → redirect تلقائي للـ auth

### 5. صفحة الرحلة `/journey`
- Dashboard تحليلي متقدم: منحنى الوعي SVG، خلاصة ذكية، timeline 28 يوم
- ألوان مختلفة لحالات الوعي (ظل/هدية/أفضل احتمال)
- Metrics واضحة: نسبة التقدم، أيام مكتملة، متوسط الوعي

### 6. الهوية البصرية
- تصميم متسق عبر كل الصفحات — لوحة ألوان ذهبية/داكنة
- خطوط عربية مختارة بعناية (Amiri)
- RTL مطبّق بشكل صحيح في كل الأماكن

## الجيد (يعمل بشكل سليم)

### 1. الصفحة الرئيسية
- آية اليوم معروضة بشكل جميل
- خطوات الرحلة مرئية (تنفس → تأمل → دوّن → أدرك → أتمم)
- زر "واصل رحلتك" واضح ويوجه لـ `/program/day/1`
- صورة خلفية الصحراء تعطي أجواء مناسبة

### 2. صفحة التسعير `/pricing`
- 4 باقات واضحة (عيدية 28, شهري 82, سنوي 820, VIP 8200)
- مكوّن تفعيل الكود يعمل بشكل سليم مع رسائل خطأ واضحة
- التوجيه التلقائي بعد التفعيل الناجح

### 3. صفحة الحساب `/account`
- عرض إحصائيات مصحّح: أيام مكتملة، streak، متوسط الحضور
- حالة الاشتراك مع اسم الباقة
- تفاصيل الحساب (بريد، تاريخ التسجيل، الباقة)
- روابط سريعة للرحلة والبرنامج
- زر تسجيل خروج يعمل

### 4. صفحة التأمل `/reflection`
- نموذج متكامل: ملاحظة + إدراك + تمعّن (3 أعمدة)
- اختيار حالة الوعي (ظل/هدية/أفضل احتمال)
- Meaning Engine — خاصية ذكية تعطي بصيرة وسؤال مقترح
- حفظ مزدوج (reflections + answers) مع تعامل سليم مع الأخطاء

### 5. الكتاب `/book` والمسبحة `/tasbeeh`
- التحقق من الاشتراك يعمل بشكل صحيح (بعد الإصلاح)
- المشترك السنوي يحصل تلقائياً على الكتاب والمسبحة
- Paywall واضح مع رابط WhatsApp للاشتراك

## الناقص (يحتاج إضافة أو تحسين)

### 1. 🔴 روابط التنقل المكسورة في Bottom Nav
**المشكلة الأكبر:** يوجد نظامان متناقضان للتنقل السفلي:
- `MobileBottomNav` → يشير إلى: الرئيسية، البرنامج، الرحلة `/journey`، التأمل `/reflection`، حسابي
- `BottomNav` (stitch) → يشير إلى: رحلتي `/journey`، التقدم `/progress`، دفتري `/journal`، حسابي

**النتيجة:** الصفحة الرئيسية تستخدم `BottomNav` (stitch)، بينما صفحات أخرى قد تستخدم `MobileBottomNav`. المستخدم يُفقد بين نظامين.

**التوصية:** توحيد نظام التنقل السفلي في نظام واحد متسق عبر كل الصفحات.

### 2. ✅ صفحة `/program/day/[id]` موجودة وتعمل
الملف `src/app/program/day/[id]/page.tsx` موجود ويعمل بشكل سليم — يعرض `DailyJourney` مع بيانات من API + JSON المحلي. والمسار القديم `/day/[dayId]` يُعيد التوجيه تلقائياً إلى `/program/day/`.

### 3. 🟡 رسالة Paywall غير محدّثة
الـ Paywall يعرض: "اشترك بـ 82 ريال وتحصل على الكتاب كاملاً + تطبيق تمعّن سنة كاملة"
و WhatsApp يرسل: "تم تحويل مبلغ 280 ريال" — وهذا مبلغ برنامج رمضان القديم.

**التوصية:** تحديث رسالة WhatsApp في `Paywall.tsx` لتتوافق مع الأسعار الحالية (28/82/820).

### 4. 🟡 لا يوجد Onboarding بعد تفعيل الكود
بعد تفعيل الكود في `/pricing` → يُوجّه المستخدم مباشرة إلى `/program` بدون أي رسالة ترحيب أو شرح لكيفية البدء.

**التوصية:** إضافة صفحة success/onboarding بسيطة بعد التفعيل.

### 5. 🟡 صفحة `/progress` بخلفية داكنة مختلفة
صفحة `/progress` تستخدم `bg-[#15130f]` (الثيم الداكن القديم) بينما صفحة `/program` تستخدم الثيم الذهبي الفاتح الجديد (`tm-shell`, `tm-card`).

**التأثير:** المستخدم ينتقل من ثيم فاتح إلى داكن فجأة عند الضغط على "عرض سجل التأملات" من صفحة البرنامج.

### 6. 🟡 صفحة التأمل `/reflection` بثيم فاتح مختلف
صفحة التأمل تستخدم `tm-shell` (الثيم الذهبي الفاتح) بينما باقي الصفحات المرتبطة (`/journey`, `/journal`) تستخدم الثيم الداكن.

### 7. 🟡 لا يوجد تعامل مع حالة انتهاء برنامج رمضان
`RAMADAN_PROGRAM_ACTIVE = true` في `appConfig.ts` لكن `RAMADAN_ENDS_AT_ISO = "2026-03-29"` — أي بعد يومين. لا يوجد منطق واضح يتعامل مع انتهاء البرنامج تلقائياً.

### 8. 🟡 waitlist يُحفظ في localStorage فقط
في الصفحة الرئيسية (عند `ramadanClosed`), البريد المسجل للـ waitlist يُحفظ فقط في localStorage — لا يُرسل لأي API أو قاعدة بيانات.

**التأثير:** لا يمكن الوصول لقائمة المنتظرين من أي مكان آخر.

## الخلل (bugs)

### 1. ✅ تم إصلاحه — NaN في متوسط الحضور
كان `awareness_logs.level` يُعامل كرقم بينما هو string. تم إصلاحه بـ mapping: `present→3, tried→2, distracted→1`.

### 2. ✅ تم إصلاحه — 0 أيام مكتملة دائماً
كان يعتمد على جدول `reflections` بدل `progress`. تم إصلاحه باستخدام `/api/program/progress`.

### 3. ✅ تم إصلاحه — الكتاب غير متاح للمشتركين السنويين
كانت صفحة `/book` لا تتحقق من `subscription_tier`. تم إضافة الشرط.

### 4. ✅ تم إصلاحه — روابط taamun-mvp.vercel.app القديمة
تم تحديث 3 ملفات مصدرية + `.env.local` لاستخدام `taamun.com`.

### 5. ⚠️ AwarenessMeter يستخدم "aware" بينما awareness_logs يخزن "present"
في `DayExperience.tsx`، الخيارات هي: `aware | tried | distracted`
لكن في AccountClient (بعد الإصلاح)، الـ mapping يتوقع: `present | tried | distracted`
وفي `/api/awareness-log/route.ts`، القيم المحفوظة هي: `shadow | gift | best_possibility` → mapped إلى `distracted | tried | present`

**التأثير:** AwarenessMeter يرسل "aware" للـ DB بينما الكود يتوقع "present". قد يسبب عدم تطابق في الإحصائيات.

**التوصية:** توحيد أسماء المستويات عبر كل المكونات.

### 6. ⚠️ تناقض في أسماء الثيم بين المكونات
بعض الصفحات تستخدم CSS variables (`text-on-surface`, `bg-background`, `text-primary`) والبعض يستخدم ألوان مباشرة (`text-[#e8e1d9]`, `bg-[#15130f]`). هذا ليس bug وظيفي لكنه يصعّب الصيانة.
