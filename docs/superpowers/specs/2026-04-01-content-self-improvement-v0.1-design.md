# النظام الثاني: تحسين ذاتي للمحتوى — الإصدار 0.1

**التاريخ:** ٣ رمضان ١٤٤٧ هـ (1 أبريل 2026)
**الإصدار:** 0.1 — الأساس
**يعتمد على:** النظام الأول (تحليل الأنماط) — مُنجز

---

## الرؤية

القرآن ثابت. لكن أي آية تظهر لمن ومتى — هذا يتطور. مثل الذكاء الاصطناعي مفتوح المصدر: النموذج يتعلم من البيانات ويحسّن نفسه.

حلقة التحسين:
```
إجابات → أنماط (✅) → حي + حالة وعي → محتوى أدق → تمعّن أعمق → أنماط أغنى → ...
```

## الإطار: ٩ أحياء × ٣ حالات وعي

من كتاب "مدينة المعنى بلغة القرآن":

### الأحياء التسعة (محاور الحياة)

| # | الحي | الظل (كيف) | الهدية (عندي) |
|---|------|------------|--------------|
| 1 | الهوية | أبحث عن تعريف خارجي | أعرف من أكون |
| 2 | العلاقات | أحتاج الآخر ليكمّلني | أقترب من امتلاء |
| 3 | التوسّع | أهرب بالتجارب | أتوسّع لأنني حيّ |
| 4 | البناء | أبني من خوف الفشل | ألتزم من طمأنينة |
| 5 | الجمال | أستهلك الجمال | أعيش الجمال |
| 6 | العائلة | أحمل لأنني خائف | أحمل لأنني اخترت |
| 7 | الروح | أبحث عن السكينة | الصمت يحدث |
| 8 | المال | المال يملكني | المال يخدم المعنى |
| 9 | العطاء | أعطي لأُحَب | أعطي لأنني ممتلئ |

### حالات الوعي الثلاث

- **الظل** — تعيش من داخل أفكارك بدون أن تراها = لغة "كيف"
- **الهدية** — لحظة ترى فيها نفسك = لحظة الانتقال
- **أفضل احتمال** — الوعي يصبح طبيعتك اليومية = لغة "عندي"

### اكتشاف الحي والحالة

يتم عبر AI (Anthropic Claude) — نفس استدعاء المحلل الليلي الموجود. البرومبت يُوسَّع ليُرجع `district` (1-9) و `awareness_state` بالإضافة للحقول الحالية.

## المكونات

### 1. توسيع المحلل الليلي

**الملف:** `src/app/api/cron/analyze-patterns/route.ts` (تعديل)

التعديلات:
- البرومبت يطلب من AI تحديد الحي (1-9) وحالة الوعي
- AI يُرجع حقلين إضافيين: `district` (1-9) و `awareness_state` ("shadow" | "gift" | "potential")
- يُرجع أيضًا: `custom_question` (سؤال مخصص لليوم التالي) و `companion_verse` (آية مرافقة)

الحقول الجديدة في الاستجابة:
```json
{
  "district": 2,
  "awareness_state": "shadow",
  "custom_question": "ما الذي تبحث عنه في الآخر اليوم — وهل هو شيء تملكه أنت أصلاً؟",
  "companion_verse": "وَمِنْ آيَاتِهِ أَنْ خَلَقَ لَكُم مِّنْ أَنفُسِكُمْ أَزْوَاجًا لِّتَسْكُنُوا إِلَيْهَا",
  "companion_verse_ref": "الروم: ٢١"
}
```

### 2. توسيع جدول `pattern_insights`

**الملف:** `supabase/migrations/20260401_content_improvement.sql` (جديد)

```sql
ALTER TABLE pattern_insights
  ADD COLUMN district INT CHECK (district BETWEEN 1 AND 9),
  ADD COLUMN awareness_state TEXT CHECK (awareness_state IN ('shadow', 'gift', 'potential')),
  ADD COLUMN custom_question TEXT,
  ADD COLUMN companion_verse TEXT,
  ADD COLUMN companion_verse_ref TEXT;
```

### 3. بطاقة الآية المرافقة

**الملف:** `src/components/CompanionVerse.tsx` (جديد)

مكوّن client يعرض آية مرافقة شخصية:
- يجلب من `pattern_insights` لليوم السابق (نفس نمط DailyHint)
- بطاقة بتصميم مختلف عن الآية الأصلية (حدود ذهبية خفيفة)
- قابلة للإخفاء
- تظهر بعد الآية الأصلية في `DayExperience`

### 4. سؤال مخصص في DayExperience

**الملف:** `src/components/CustomQuestion.tsx` (جديد)

- يجلب `custom_question` من `pattern_insights` لليوم السابق
- يظهر بجانب سؤال التمعّن الأصلي (لا يستبدله)
- بطاقة صغيرة بعنوان "سؤال لك"
- قابلة للإخفاء

### 5. تغذية المرشد

**الملف:** `src/app/api/guide/route.ts` (تعديل)

- عند استدعاء المرشد، يُجلب آخر `pattern_insights` للمستخدم
- يُضاف للبرومبت: الحي الحالي، حالة الوعي، المواضيع المتكررة
- المرشد لا يذكر هذه المعلومات مباشرة — يستخدمها لتعميق الاستجابة

### 6. تعديل DayExperience

**الملف:** `src/components/DayExperience.tsx` (تعديل)

إضافة `CompanionVerse` و `CustomQuestion` في الترتيب:
```
DailyHint (تلميح — موجود ✅)

ProgressionBadge
VerseBlock (الآية الأصلية)
CompanionVerse (آية مرافقة — جديد)
CustomQuestion (سؤال مخصص — جديد)
HiddenLayer
...باقي المكونات
```

## حدود الإصدار 0.1

- **الآيات المرافقة يختارها AI** من سياق إجابات المستخدم + معرفة الأحياء — لا يوجد جدول مرجعي ثابت (يُضاف في 0.2)
- **لا يُستبدل المحتوى الأصلي** — كل شيء إضافي وقابل للإخفاء
- **اكتشاف الحي عبر AI فقط** — لا keyword matching (أدق لكن أبطأ)
- **لا تخصيص للتمارين** — يبقى التمرين الأصلي (يُضاف لاحقًا)
