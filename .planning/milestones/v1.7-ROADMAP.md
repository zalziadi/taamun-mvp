## Mission

v1.6 made invisible work visible. v1.7 deepens the creator graph:
public profile pages, a "following" surface for members, smarter
discovery ranking, and an admin queue badge.

---

## Shipped phases

### Phase 1: Public creator profile
- `src/app/creator/by/[userId]/page.tsx` — SSR, anon-readable, lists every published journey of a creator, pulls display name from the most-subscribed entry, shows total-subscribers aggregate, includes the existing follow button
- `generateMetadata` + OG (`type: profile`) for shareability

### Phase 2: Following tab in /account
- `src/app/api/creator/follows/route.ts` — per-user feed: all followed creators + their latest published journey + total published count
- `src/components/FollowingList.tsx` — compact card, quiet when empty, links into `/creator/by/[userId]` + `/journey/[slug]`
- Wired into `/account/AccountClient.tsx` with `dynamic(ssr: false)`

### Phase 3: /discover ranking + top section
- `src/app/discover/page.tsx` — primary sort `subscriber_count DESC`, tiebreaker `created_at DESC`; split into "الأكثر اشتراكاً" (top 3 with traction) + "الأحدث" (everything else by newest); empty state preserved
- Shared `JourneyRow` component prevents card-markup drift

### Phase 4: Admin moderation queue badge
- `src/components/ModerationQueueBadge.tsx` — client-side fetch of `/api/admin/moderation`, renders an amber hero strip on `/admin` only when total > 0
- Non-admins or empty queue → renders nothing (zero noise)

---

## Non-functional compliance

- Zero new dependencies, zero new migrations
- `/creator/by/[userId]` is anon-readable (SEO-indexable) + `force-dynamic`
- `FollowingList` lazy-loaded with `ssr: false` (no hydration cost for the common empty state)
- Discover sort uses DB-level `.order()` (no client sort overhead)
- Badge is a single fetch behind admin gate; silent 401 when the viewer isn't an admin
