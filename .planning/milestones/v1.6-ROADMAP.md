## Mission

v1.5 closed loops around community + creator surfaces. v1.6 makes the
invisible work visible:

1. **Users see** the invite credits they've earned
2. **Creators see** their follower graph + auto-push on new journeys
3. **Founder sees** a single daily email summarizing the moderation queue

1. **Credits UI location:** inline inside the existing InviteShare widget (subtle +X days badge)
2. **Follow default:** off (explicit opt-in)
3. **Mod digest cadence:** daily 08:00 UTC (11 KSA)
4. **Mod digest channel:** email (ADMIN_EMAIL via Resend) — silent when unconfigured or queue empty

## Shipped phases

### Phase 1: Invite credits visibility
- `src/app/api/invite/my-rewards/route.ts` — per-user breakdown (earnedDays, totalInvited, rewardedCount, redeemed[])
- `src/components/InviteShare.tsx` — badge shows `+{rewarded * 30} يوم رصيد مكتسب` alongside existing "استخدمه X شخص · X اشترك"

### Phase 2: Creator follow system
- `supabase/migrations/20260420300000_creator_follows.sql` — `creator_follows` table with RLS (follower owns row, creator can read their list), UNIQUE + self-follow CHECK, 2 indexes
- `src/app/api/creator/[creatorId]/follow/route.ts` — GET/POST/DELETE toggle; GET returns `{following, followerCount}` with count visible only to the creator themselves
- `src/components/CreatorFollowButton.tsx` — compact client toggle, auth-redirect on 401, loading placeholder
- `/journey/[slug]` — follow button rendered next to subscribe
- `/api/creator/journeys/[slug]` PATCH — on draft→published transition, fans out a push to every follower (`sendPushToUser` per row, fire-and-forget; failure never blocks the PATCH)

### Phase 3: Daily moderation summary email
- `src/app/api/cron/mod-digest/route.ts` — counts flagged rows across 4 UGC surfaces; silent no-op on empty queue or missing RESEND_API_KEY/ADMIN_EMAIL; sends an RTL Arabic HTML email with a CTA button to `/admin/moderation`
- `vercel.json` — new cron `{"path":"/api/cron/mod-digest","schedule":"0 8 * * *"}`

## Non-functional compliance

- Zero new npm dependencies
- Every new surface RTL Arabic
- All writes guarded by auth or admin gate
- Push + email fail silently — no observable user-facing error when env isn't set
- `creator_follows` CHECK prevents self-follow at the DB level
