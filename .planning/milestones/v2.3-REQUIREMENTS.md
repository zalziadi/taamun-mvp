# Requirements — v2.3 (archived)

Snapshot of v2.3 requirements as shipped 2026-04-20.

---

## Mission

Alert thresholds on top of v2.2 ops_snapshots. Email digest when any
active rule crosses its threshold. Debounced 12h per rule.

---

## Must-have ✅

- ✅ `ops_alerts` migration + 3 seeded defaults
- ✅ CRUD API at `/api/ops/alerts`
- ✅ Daily `/api/cron/check-alerts` with 12h debounce + Resend email
- ✅ `/admin/ops/alerts` UI

---

## Out of scope

- Push notification channel (email only for v2.3)
- Per-rule cooldown override
- Multi-recipient fan-out

---

## Open questions — resolved

1. **Operators:** ✅ `>`, `<`, `>=`, `<=`, `==`
2. **Channel:** ✅ email via Resend
3. **Debounce:** ✅ 12h per rule
4. **Seeds:** ✅ 3 defaults on migration
