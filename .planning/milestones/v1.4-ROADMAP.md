## Mission

v1.3 gave users **channels** to spread Taamun. v1.4 builds **places** where users come back together — community space inside the app + creator tools so trusted voices can publish their own journeys.

Close the loop: acquisition (v1.3) → retention (v1.1) → depth (v1.2) → **belonging**.

## Open questions — resolved

1. **Credit event:** first successful payment → triggers +30 days
2. **Thread identity:** both — real name OR pseudonym per post
3. **Creator approval:** any VIP can publish (tier is the gate); auto-flag catches URL/handle injection
4. **OG fonts:** Amiri (body) + Noto Naskh Arabic (bold) — loaded at request time in ImageResponse
5. **Year-end recap:** both Dec 31 (Gregorian) + Muharram 1 (Hijri) — single daily cron checks both anchors

## Shipped phases

### Phase 1: Invite Reward Credit
Apply +30 days to both inviter and invitee on first successful subscription.

**Deliverables:**
- `supabase/migrations/20260420000000_membership_expiry.sql` — adds `profiles.expires_at` + partial index
- `src/lib/invite-rewards.ts` — `applyInviteReward(admin, inviteeUserId)` extends membership for both parties; idempotent via `invite_redemptions.rewarded` flag with race-safe atomic guard
- `src/app/api/invite/apply-reward/route.ts` — admin POST protected by `ADMIN_MIGRATION_KEY` / `SUPABASE_SERVICE_ROLE_KEY`; designed to be called from Stripe/Salla/Tap webhooks or manual admin trigger

### Phase 2: In-App Threads
Lightweight public discussions attached to a day or verse.

**Deliverables:**
- `supabase/migrations/20260420100000_threads.sql` — `threads` + `thread_replies` tables, RLS (public read published, owner CRUD own), auto-increment `reply_count` trigger
- `src/lib/thread-moderation.ts` — shared moderation heuristic (URL/domain/handle → flagged)
- `src/app/api/threads/route.ts` — GET (filter by anchor) + POST (auth + moderate)
- `src/app/api/threads/[id]/replies/route.ts` — GET (oldest first) + POST (auth + moderate + thread existence check)
- `src/app/threads/[id]/page.tsx` — SSR thread + replies, `generateMetadata` for OG
- `src/components/ThreadReplyForm.tsx` — client compose (displayName + body), 401 → /auth redirect, flagged/published status messaging

### Phase 3: Creator Mode MVP
VIPs publish their own 7- or 14-day mini-journeys.

**Deliverables:**
- `supabase/migrations/20260420200000_creator_journeys.sql` — `creator_journeys` + `creator_journey_days` + `creator_journey_subscriptions` tables; RLS (public read published, creator CRUD own, subscriber owns progress); subscriber-count trigger
- `src/lib/creator-slug.ts` — slug builder with ASCII fallback + random suffix
- `src/app/api/creator/journeys/route.ts` — GET own list + POST create (VIP gate via `isVipTier(profile.subscription_tier)`, collision-retry on slug)
- `src/app/api/creator/journeys/[slug]/route.ts` — GET public/owner view + PATCH (publish gate requires all days present + auto-flag on text) + DELETE
- `src/app/api/creator/journeys/[slug]/days/route.ts` — POST upsert by (slug, day_number) + DELETE
- `src/app/api/creator/journeys/[slug]/subscribe/route.ts` — POST subscribe (idempotent) + PATCH complete_day advancing current_day
- `src/app/discover/page.tsx` — public catalog of published journeys
- `src/app/journey/[slug]/page.tsx` — public subscribe / continue view with day previews
- `src/components/JourneySubscribeButton.tsx` — auth-gated subscribe
- `src/app/creator/page.tsx` — creator dashboard (own list + new-journey form, VIP gate)
- `src/app/creator/[slug]/page.tsx` — editor with day tabs, publish/unpublish/delete flow

### Phase 4: OG Image Generation
Server-rendered Arabic calligraphy preview for `/shared/[slug]`.

**Deliverables:**
- `src/app/shared/[slug]/opengraph-image.tsx` — Next.js `next/og` ImageResponse at 1200×630; fetches Amiri Regular + Noto Naskh Arabic Bold from Google Fonts CSS2 at request time (canonical ImageResponse pattern; `next/font` is not applicable inside ImageResponse); dynamic font sizing for long vs. short quotes

### Phase 5: Year-end Recap Email Cron
Annual recap email on both Gregorian year-end and Hijri new year.

**Deliverables:**
- `src/lib/year-recap.ts` — `isGregorianYearEnd` + `isHijriNewYear` (Umm al-Qura via `Intl.DateTimeFormat`); `currentYearLabel` for template suffixing
- `src/app/api/cron/year-recap/route.ts` — daily cron that fires only on anchor days; queues one email per eligible user (≥ 90 days since `user_progress.started_at`) per variant per year; idempotency via `email_queue.template = year_recap_{label}` + 350-day lookback
- `vercel.json` — adds `{"path":"/api/cron/year-recap","schedule":"0 7 * * *"}`

## Non-functional compliance

- **Performance:** no client-bundle regressions; `/discover` + `/journey` + `/threads` are server components; only the creator editor + subscribe button + thread reply form ship client JS
- **A11y:** reuses existing `tm-card` + 3-line aesthetic; RTL throughout; submit buttons labeled in Arabic
- **SEO:** `generateMetadata` + OpenGraph on both `/journey/[slug]` and `/threads/[id]`; `/discover` is indexable with static metadata
- **Cost:** zero new dependencies; `next/og` built into Next 14
- **Privacy:** threads and journeys are opt-in public; pseudonym allowed for both
- **Moderation:** founder-review queue via `status = 'flagged'`; URL/handle auto-flag reused across threads + creator publishing
