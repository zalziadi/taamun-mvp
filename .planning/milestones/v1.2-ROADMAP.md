## Shipped phases

### Phase 2: Guide Long-term Memory (commit 4bba4be)

`guide_memory.soul_summary` evolves weekly so every guide conversation benefits.

**Deliverables:**
- `src/lib/soul-summary-generator.ts` — Claude-powered narrative summary
  - Inputs: up to 30 recent reflections + awareness pattern + existing summary
  - Outputs: 3-5 sentence Arabic summary + 3-5 recurring themes
  - Merges intelligently with prior summary (doesn't replace)
- DB migration: extends `guide_memory` with `themes[]`, `soul_summary_updated_at`, `soul_summary_token_count`
- `/api/cron/refresh-soul-summaries` — Sunday 3 AM UTC (6 AM KSA)
  - Iterates active subscribers with ≥3 reflections
  - Skips if < 7 days old AND not on milestone day (14/28/60)
  - Returns per-user results + total tokens

**Impact:** the guide prompt already reads `soul_summary` (v1.0 code). This phase populates it automatically. Every guide conversation now references the user's cumulative journey.

**Cost:** ~$0.002 per user per week.

---

### Phase 1: Reflection Theme Clustering (commit 45837f3)

Extract recurring themes from the user's reflections.

**Deliverables:**
- `src/lib/reflection-clustering.ts`
  - OpenAI `text-embedding-3-small` for each reflection (batch call)
  - Cosine-similarity agglomerative clustering (threshold 0.6)
  - Claude labels clusters with Arabic theme names + 3-5 keywords
- DB migration: `reflection_themes` table with RLS owner-only SELECT
- `/api/reflections/themes` — user read endpoint
- `/api/cron/refresh-themes` — 1st of month 4 AM UTC (7 AM KSA)
  - For each active subscriber with ≥5 reflections
  - Skips if refreshed within 25 days
  - Replaces prior themes atomically (delete + insert)

**Cost:** ~$0.008 per user per month.

### Phase 3: Voice Journaling (commit 0df7238)

Extend the journaling surface to voice.

**Deliverables:**
- `src/components/VoiceReflection.tsx` — drop-in component
  - `MediaRecorder` API (webm/opus, 90s max aligned with 2 MB STT limit)
  - State machine: idle → requesting → recording → processing → done/error
  - Graceful fallback for unsupported browsers
  - RTL Arabic UI
  - Reuses existing `/api/voice/stt` (Munsit) and `/api/reflections` (upsert)

**Placement:** component is ready — integration into DayExperience / reflection / journal pages deferred to product UX decision.

---

### Phase 4: Accessibility CI (commit 0df7238)

Automate the a11y audit so regressions are caught in CI.

**Deliverables:**
- `.lighthouserc.json` — targets `/`, `/welcome`, `/pricing`
  - Assertions: A11y ≥ 0.95 (error), SEO ≥ 0.95 (warn), Best Practices ≥ 0.8 (warn)
  - 2 runs per URL, desktop preset
- `.github/workflows/lighthouse.yml` — runs on push to main touching `src/**`, `public/**`, or `.lighthouserc.json`
  - Waits 60s for Vercel deploy propagation
  - Uses `@lhci/cli@0.14.x`
  - Manual `workflow_dispatch` also enabled

**Impact:** any PR that breaks A11y fails CI before merge.
