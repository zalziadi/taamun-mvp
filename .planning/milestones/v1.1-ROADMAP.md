## Shipped phases

### Phase 1: Email Automation (commit ae27e28)

Weekly digests + re-engagement emails, fully automated.

**Deliverables:**
- `/api/cron/queue-digests` (Saturday 4 AM UTC / 7 AM KSA)
  - Queues `weekly_digest` for active subscribers (reflection + awareness counts)
  - Queues `reengagement` for users inactive 3+ days
  - Idempotent: no duplicates within 7 days
- `weekly-digest-template.ts` тАФ Saturday summary with stats + encouragement
- `reengagement-template.ts` тАФ tiered messages (3/7/14+ days inactive)
- `send-emails` route handles the 2 new template types
- `vercel.json` registers the cron

**Design principle:** "no guilt тАФ the journey is not a race."

### Phase 2: Web Push Notifications (commit b48ff76)

Morning reminder with user-customizable time.

**Deliverables:**
- `web-push` npm package installed + VAPID keys generated
- `public/sw.js` тАФ service worker (RTL + Arabic + click-to-open)
- DB: `push_subscriptions` table (endpoint + keys + preferences + failure tracking)
- 3 API endpoints:
  - `POST /api/push/subscribe` тАФ save subscription + preferences
  - `PATCH /api/push/subscribe` тАФ update preferences
  - `POST /api/push/unsubscribe` тАФ remove by endpoint
- `/api/cron/send-push` тАФ hourly matcher, sends today's verse to users whose `morning_hour` matches current UTC hour
- `PushSetup` component in `/account` тАФ permission request + hour slider (UTCтЖФKSA conversion)
- Auto-delete 410/404 dead subscriptions
- `vercel.json`: hourly cron

**Cost:** zero (Web Push is free). Required env vars: `NEXT_PUBLIC_VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT`.
