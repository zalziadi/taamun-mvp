## Shipped phases

### Phase 1: Moderation dashboard
- `src/app/api/admin/moderation/route.ts` — GET aggregates flagged rows across `threads`, `thread_replies`, `creator_journeys`, `shared_insights`; POST moves a row to `published`/`removed`/`flagged` with admin gating
- `src/app/admin/moderation/page.tsx` — RTL Arabic dashboard with per-bucket lists and inline action buttons
- Admin card in `/admin` dashboard links to the new page

### Phase 2: Thread reply push notifications
- `src/lib/push.ts` — shared `sendPushToUser()` helper with VAPID setup + expired-subscription cleanup
- `src/app/api/threads/[id]/replies/route.ts` — fires a push to the thread author on a new published reply (skipped when the author replies to themselves); failure is logged but never fails the reply creation

### Phase 3: Creator analytics
- `src/app/api/creator/journeys/[slug]/analytics/route.ts` — owner-only: total subscribers, finished, completion-rate %, per-day completion counts, last 10 active subscribers
- `src/app/creator/[slug]/analytics/page.tsx` — visualization (stat tiles + per-day drop-off bars + recent activity)
- Editor header in `/creator/[slug]` now includes a "تحليلات" link

### Phase 4: v1.4 in-app activations
- `/discover` link added to desktop nav (`AppChrome.tsx`)
- `/creator` entry added to `/account` post-28 CTA row, VIP-gated via `planTier`
- New `src/app/threads/page.tsx` — list view accepting `?anchor_type=verse&anchor_value=...`
- `DayExperience.tsx` — "ناقش هذه الآية مع المجتمع" link at the end of every day, piping `content.verseRef` into the threads filter
