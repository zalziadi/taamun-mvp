## Shipped phases

### Phase 1: PWA Polish + Install Prompt (commit aa10b47)

Turn Taamun into an installable home-screen app.

**Deliverables:**
- `public/manifest.json` — RTL Arabic, standalone display, theme color `#5a4a35`, 2 app shortcuts (program + guide)
- `public/sw.js` extended — offline caching for static assets (cache-first) and HTML pages (network-first with cached fallback); push handlers preserved from v1.1
- `src/components/ServiceWorkerRegistrar.tsx` — registers `/sw.js` on idle (zero LCP impact); wired into `layout.tsx`
- `src/components/InstallPrompt.tsx` — dismissable banner
  - Shown day 7+
  - Re-prompts at day 28 if dismissed
  - Hides when already installed (`display-mode: standalone`)
  - iOS fallback with Add-to-Homescreen tip
  - Chromium uses `beforeinstallprompt` → native install dialog
- `layout.tsx` metadata: `manifest: "/manifest.json"`, `themeColor: "#5a4a35"`, `appleWebApp`

### Phase 2: Invite Flow (commit aa10b47)

Unique invite code per user, reward system for successful signups.

**Deliverables:**
- DB migration `invite_system.sql`:
  - `invite_codes` — one permanent code per user, public read (for landing page), owner-only write
  - `invite_redemptions` — logs inviter/invitee pairs with `rewarded` flag for subsequent credit application
  - `increment_invite_uses()` SECURITY DEFINER function for atomic counter
- `/api/invite`:
  - `GET` — get or create the user's invite code + stats (uses, rewarded count)
  - `POST` — redeem an invite code during signup (idempotent per invitee)
- `/invite/[code]` — public landing page
  - Validates code against DB (anon can read via public policy)
  - If user already logged in: redeems + redirects to `/welcome`
  - If not: shows CTA with "صديق دعاك" framing, signup link carries the code
- `src/components/InviteShare.tsx` (in `/account`):
  - Loads code via GET
  - Displays full invite URL
  - Copy button + native `navigator.share()` with fallback
  - Shows usage stats + rewarded count

**Reward model:** Free month for both inviter and invitee on successful subscription. Actual credit application deferred to post-signup conversion event (not this milestone's scope — prevents free-month fraud).

### Phase 5: Voice Tasbeeh (commit aa10b47)

Hands-free dhikr counter using free browser API.

**Deliverables:**
- `src/components/VoiceTasbeeh.tsx`:
  - Web Speech API (`SpeechRecognition` with `webkit` prefix fallback)
  - Listens in continuous mode for `ar-SA`
  - 3 phrases: سبحان الله (33) · الحمد لله (33) · الله أكبر (34) — cycle via button
  - Debounce 500ms between matches (prevents echo multi-count)
  - Auto-restart on browser-triggered `onend` (Chrome stops after silence)
  - Manual `+1` fallback if Web Speech unavailable
  - Reset button per phrase
  - Progress bar + `أتممت` celebration at target

**Placement:** component is drop-in ready. Integration into `/tasbeeh` or new standalone page deferred to product decision.

### Phase 4: Shareable Insights (commit aa10b47)

Opt-in, anonymous, SEO-indexable quote pages.

**Deliverables:**
- DB migration `shared_insights.sql`:
  - Slug as primary key
  - `content` CHECK (10 ≤ length ≤ 200)
  - Optional `attribution` (max 60 chars, null = anonymous)
  - `status` ∈ {published, flagged, removed}
  - Public read for `status = 'published'` (both anon and authenticated)
  - Owner-only write
- `/api/shared`:
  - `POST` — creates insight with auto-moderation (flags if contains URL patterns like `https://`, `www.`, `.com`, `@`)
  - `DELETE` — user removes their own insight
- `/shared/[slug]` — public page:
  - `generateMetadata` for OG preview
  - Amiri typography, minimal borders matching minimal UI aesthetic
  - CTA to `/` at bottom
  - `notFound()` for invalid/flagged/removed slugs
  - Async view count increment (fire-and-forget)

**Moderation policy:** auto-publish for clean short content; founder reviews flagged queue via direct DB access. No in-app moderation UI yet (v1.4 if needed).

### Phase 3: Year in Taamun Recap (commit aa10b47)

Personal "year in review" page, eligible after 90 days.

**Deliverables:**
- `/app/recap/page.tsx`:
  - Eligibility gate: `profiles.created_at` must be ≥ 90 days ago
  - If ineligible: beautiful "available after 90 days" message + countdown
  - If eligible: shows
    * Total reflection count
    * Total awareness log count
    * `soul_summary` from `guide_memory` (v1.2 surface)
    * Top 3 themes from `reflection_themes` (v1.2 surface)
    * CTA to `/guide` and `/insights`
  - Parallel data fetch (5 queries in Promise.all)
  - Arabic numeral conversion for display
  - OG-friendly metadata

**Rolling trigger:** per-user day 365. An optional batch email at calendar year-end is queued as v1.4 work.
