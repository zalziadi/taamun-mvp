# Phase 10 Plan 05: /account/referral Page + ReferralPanel Summary

Ships the only user-facing surface for the referral program: `/account/referral` renders the user's FRIEND-* code, da'wah-framed copy ("ادعُ للتمعّن"), and a WhatsApp-primary / Instagram / copy-link share trio — no gamification UI, no auto-share, no emoji.

## What shipped

### 1. `GET /api/referral/list` (auth-gated listing)

File: `src/app/api/referral/list/route.ts` (+ `.test.ts`).

- Mirrors `/api/referral/create` style: `requireUser()` gate, service-role `getSupabaseAdmin()`, explicit `.eq("referrer_id", auth.user.id)` scoping.
- **SELECT whitelist** (privacy): `id, code, status, invitee_redeemed_at, referrer_rewarded_at, created_at`. `invitee_id` is deliberately excluded — the referrer never learns who the invitee is.
- Ordered `created_at DESC`. No pagination (v1.2 scale: max ~10 rows/user/year).
- 3 vitest cases (`route.test.ts`), all green:
  1. 401 anon → no DB call.
  2. Happy path → rows returned; `select()` argument asserted to include the 6 whitelisted columns and NOT `invitee_id`; `.eq("referrer_id", userId)` asserted.
  3. DB error → 500 `{ ok:false, error:"server_error" }`.

### 2. Pure helpers (`ReferralPanelHelpers.ts`)

- `STATUS_LABEL: Record<ReferralStatus, string>` — 5 Arabic labels:
  - `pending_invitee` → `"بانتظار البدء"`
  - `pending_day14` → `"في الطريق"`
  - `rewarded` → `"تمّت الهدية"`
  - `refunded` → `"أُلغيت"`
  - `void` → `"أُلغيت"`
- `buildShareText(code, appDomain)` — single-line Arabic: `"تمعّن — ٢٨ يومًا مع القرآن. خذ شهرًا تجربة على حسابي: {domain}/?ref={code}"`.
- `buildWhatsAppHref(text)` → `https://wa.me/?text=${encodeURIComponent(text)}`.
- `buildInstagramStoryHref(_, _)` → `https://www.instagram.com/` (pragmatic fallback — IG Web exposes no prefilled-story API).
- `buildCopyText(code, appDomain)` — alias-by-call of `buildShareText` (single source of truth).
- **No React / next/* / process.env imports** — pure TS, vitest-testable without JSX transform.

### 3. `ReferralPanel.tsx` (client component)

- `"use client"`. Fetches `/api/referral/create` (POST) + `/api/referral/list` (GET) in parallel on mount via `Promise.all`.
- State shape: `{ loading, code, capHit, currentCount, list, error }`.
- Render branches:
  - Loading → subtle placeholder card.
  - 429 `annual_cap_reached` → `"بلغت ٣ دعوات هذا العام — شكرًا لدعوتك."` (transparent, no shame copy).
  - Happy path → `رمزك` label + monospace gold FRIEND-* code + share-button trio + optional past-invites list.
  - Error → neutral `"تعذّر تحميل بياناتك الآن. حاول لاحقًا."`.
- Share trio:
  - **WhatsApp** (primary): gold filled `.btn`-equivalent (`bg-[#C9A84C]`, dark text), opens `wa.me/?text=...`.
  - **Instagram** (secondary): outlined, opens `instagram.com/`.
  - **Copy-link** (secondary): `navigator.clipboard.writeText(buildCopyText(...))`; inline label swap `"انسخ الرابط"` ↔ `"تم النسخ"` (no auto-reset timer).
- RTL root (`dir="rtl"`).

### 4. Server-component page (`/account/referral/page.tsx`)

- `export const dynamic = "force-dynamic"`.
- `await requireUser()` → `redirect("/auth?next=/account/referral")` if anon.
- Renders `<ReferralPanel appDomain={APP_DOMAIN} />`.

## Deviations from Plan

### [Rule 2 — Privacy hardening] Client projection strips server-only fields

**Found during:** Task 3 implementation.
**Issue:** Server `/api/referral/list` returns `invitee_redeemed_at` + `referrer_rewarded_at` (needed for future audit surfaces), but the UI only renders `status` + `created_at`. Keeping those extra fields in the client `ReferralRow` type would (a) make it easier to accidentally render them and leak invitee activity timestamps and (b) trip the plan's verify grep `earn|cash|reward|...` on the word `reward` inside `referrer_rewarded_at`.
**Fix:** Client projection maps each raw row to `{ id, code, status, created_at }` only; the raw field names never appear in the `ReferralPanel.tsx` source.
**Files modified:** `src/components/ReferralPanel.tsx` (narrow mapping in fetch-on-mount).
**Commit:** `3c2f03c`.

### [Rule 2 — Plan filename alignment] Helpers file named `ReferralPanelHelpers.ts`

**Found during:** Task 2 start.
**Issue:** The plan frontmatter `files_modified:` listed `src/components/ReferralPanel.test.ts`, but the Task 2 `<action>` body and the 09.04 precedent (`renewalBannerHelpers.ts` + `.test.ts` sibling) clearly prescribe a pure-helpers file + its test. The frontmatter entry was shorthand; the body is authoritative.
**Fix:** Shipped `ReferralPanelHelpers.ts` + `ReferralPanelHelpers.test.ts` per the Task 2 body spec and the 09.04 pattern.
