## Runtime decision: edge (not nodejs)

The orchestrator's critical_rules requested `runtime = 'nodejs'`. The agent's file comment (lines 45-48) explains:
> Edge runtime is mandatory: next/og's ImageResponse is only supported on the edge runtime in Next.js 14.2.18 (verified against STACK.md).

This is verified correct — `next/og` in Next 14.2 requires edge. The Vercel 2026 guidance to "prefer Fluid Compute over edge" is aspirational; when a framework API only supports edge, edge wins. This is documented as a known trade-off; upgrade to Next 15+ would unlock nodejs runtime for ImageResponse.
