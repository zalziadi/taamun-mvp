## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Node ESM cannot import src/lib/analytics/server.ts**
- **Found during:** Task 1 initial run
- **Issue:** `server.ts` imports `./events` (no .ts extension) — Node 25 ESM refuses to resolve extension-less relative imports even with `--experimental-strip-types`
- **Fix:** Inlined a shape-equivalent `emitEvent()` in the harness (20 LOC). Mirrors REAL server.ts behavior: prefix-only property guard + fire-and-forget fetch to PostHog capture URL. The fetch interceptor captures into `postHogSink` exactly as if the real module had been called.
- **Why not add a loader:** NFR-08 forbids new deps (ts-node, tsx, esbuild-register all qualify).
- **Impact:** Zero. The REAL `generateUniqueFriendCode` / `creditOneReferral` / `FRIEND_CODE_REGEX` are imported and exercised — only the thin analytics transport is mirrored.
- **Files modified:** scripts/test-phase-10-integration.mjs
- **Commit:** 7b5b4f0

**2. [Rule 1 - Bug] Fake Supabase returned `undefined` instead of `null` for unset columns**
- **Found during:** Task 1 scenario 2 run
- **Issue:** The fake omits undeclared columns rather than emitting `null`. Scenario 2's activate mirror checked `referralRow.invitee_id !== null` — `undefined !== null` is true, triggering the `code_already_redeemed` 409 branch on a fresh mint.
- **Fix:** (a) Changed the mirror's check to `invitee_id != null` (catches both null and undefined) — real PG would return null, so behavior is identical on prod. (b) Added explicit `invitee_id: null` to the insert call so reads downstream see the expected value.
- **Impact:** Zero on prod behavior. Test-only shape fix.
- **Files modified:** scripts/test-phase-10-integration.mjs
- **Commit:** 7b5b4f0

**3. [Rule 2 - Missing] Scenario 7 cumulative sink**
- **Found during:** Task 1 final run
- **Issue:** Scenarios 4-6 each call `clearSink()` at their top to verify per-scenario event counts cleanly. By scenario 7 the current sink is empty (scenario 6 never emits, since cap blocks creation).
- **Fix:** Added `postHogSinkAll` — a separate cumulative sink the interceptor also writes to, never cleared. Scenario 7 uses it for the end-of-run ANALYTICS-07 sweep.
- **Impact:** Cleaner; no behavioral change.
- **Files modified:** scripts/test-phase-10-integration.mjs
- **Commit:** 7b5b4f0
