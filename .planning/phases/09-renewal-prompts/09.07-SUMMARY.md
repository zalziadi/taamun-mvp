## Deviations from Plan

**None with respect to scope.** Three minor mechanical adjustments documented here for traceability:

1. **Harness path choice** — plan frontmatter specifies `scripts/test-phase-09-integration.mjs` (repo root of `scripts/`); Phase 7/8 used `scripts/verify/phase-XX-integration.mjs`. Followed the plan's frontmatter path verbatim as the authoritative specification.

2. **Scenario A day math** — initial expiry `now + 6d + 5000ms` made `Math.ceil(6.00005) = 7`, failing the `daysRemaining === 6` assertion on the first run. Adjusted to `now + 5.5d` so `Math.ceil(5.5) = 6` deterministically regardless of ms-drift between write and compute. Pure test-fixture tweak — no production code change.

3. **Node 22 TS-stripping** — Scenarios C, E, F were written with `import("../src/lib/...ts")` knowing Node 22 would strip TS inline. Added a `.catch()` fallback to inline-mirror the helpers for older Node, so the harness works on Node 18+. `package.json` `engines` is unspecified; Vercel build uses Node 20 (also supports TS stripping via `--experimental-strip-types`). Confirmed working on Node v25.8.1 locally.
