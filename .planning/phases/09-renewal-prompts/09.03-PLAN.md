<risks_and_mitigations>
- **Pitfall 17 (multi-gateway mismatch) — regression vector:** The `.is("original_gateway", null)` guard ensures first-gateway-wins. Without it, a user switching gateways would mis-route renewal.
- **Pitfall — webhook retry storm:** Try/catch + non-2xx avoidance means a transient DB failure on the gateway-tag write does NOT cause the payment provider to retry the whole payment webhook. Critical for Salla/Stripe which aggressively retry.
- **Stripe webhook — customer already exists:** If a user upgrades their Stripe plan, the webhook may fire again. The `.is(null)` guard keeps their original_gateway stable.
- **Activation route double-call (user pastes code twice):** existing `used_by` conflict (409) already returns early; our write path runs only on first success.
</risks_and_mitigations>
