## Schema Introspection Findings (for Plan 09.07)

The plan's `<interfaces>` block made column-name assumptions that do NOT match the real migrations. Plan 09.07 fixtures MUST use these actual names:

| Table | Plan assumed | Actual column (verified) | Source migration |
|-------|--------------|--------------------------|------------------|
| `email_queue` | `type` | **`template`** | `20260328000000_email_queue.sql` |
| `email_queue` | `sent_at` for dedup | **`created_at`** (sent_at is NULL until cron flushes; see decision below) | same |
| `push_subscriptions` | `last_notified_at` | **`last_sent_at`** | `20260418100000_push_subscriptions.sql` |

**email_queue.template values observed in codebase** (via grep of `.from("email_queue").insert`):
- `"activation"` (default), `"expiry_warning"`, `"expired"`, `"reengagement"`, `"weekly_digest"`, `"weekly_challenge"`, `"completion"`

Dedup list used by this helper: `["expiry_warning", "renewal", "expired"]`. `"renewal"` is forward-compatibility — no writer inserts that template today, but if Plan 09.07+ introduces a dedicated renewal email, it'll already be deduped.

**Why `created_at` instead of `sent_at` for dedup:**
`email_queue.sent_at` is NULL until the 5-min email cron (`/api/cron/send-emails/route.ts`) flushes the row. Using `sent_at` would let a banner render on top of an email-about-to-arrive. Using `created_at` captures the nudge the moment it's queued — matching user-perceived nudge count.

## Decisions Made

1. **Dedup on `created_at`, not `sent_at`** — see Schema Introspection. Preserves the "1 nudge per 24h" contract even during the 5-min queue-to-flush gap.
2. **Broadened template dedup list** to `['expiry_warning', 'renewal', 'expired']` — if `expired` fired this morning (user already lapsed), the banner suppresses.
3. **Runtime gateway narrowing** — accept only `'salla' | 'tap' | 'stripe' | 'eid_code'`. Any other value → `{show: false, reason: 'no_gateway'}`. Defense-in-depth against backfill gaps.
4. **Single commit instead of RED + GREEN + REFACTOR split** — the RED phase is implicit (import resolves to nothing, test file errors on import), so splitting the commit would add noise without signal. REFACTOR not needed — implementation is ~60 lines of straight-line logic.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Schema drift: `email_queue.type` does not exist**
- **Found during:** Task 1 (writing the query)
- **Issue:** Plan 09.05 line 120 uses `.eq("type", ...)` — the actual migration (`20260328000000_email_queue.sql`) names the column `template`. Build would compile (no FK check on string columns) but the query would match zero rows, silently breaking dedup.
- **Fix:** Changed `.eq("type", ...)` to `.in("template", [...])` with the verified template slugs.
- **Files modified:** `src/lib/renewal/shouldShow.ts`
- **Verification:** 9/9 tests pass + grep of actual inserters (`manage-subscriptions/route.ts` line 56) confirms the column name.
- **Committed in:** `acfe909`

**2. [Rule 1 - Bug] Schema drift: `push_subscriptions.last_notified_at` does not exist**
- **Found during:** Task 1 (writing the push dedup query)
- **Issue:** Plan 09.05 line 43 references `last_notified_at`. Actual migration (`20260418100000_push_subscriptions.sql`) uses `last_sent_at`. Query would return zero rows.
- **Fix:** Used `last_sent_at` in both the select and the gte filter.
- **Files modified:** `src/lib/renewal/shouldShow.ts`, `src/lib/renewal/shouldShow.test.ts` (test scenario 8 asserts on `last_sent_at`).
- **Verification:** Test 8 passes with simulated 10h-ago push.
- **Committed in:** `acfe909`

**3. [Rule 2 - Missing Critical] Dedup-on-created_at semantics**
- **Found during:** Task 1 (while fixing deviation 1)
- **Issue:** The `email_queue` row is inserted BEFORE it's sent — `sent_at` is populated by the cron ~5 min later. Using `sent_at` would let the banner render for 5 min on top of an email about to land. That's the exact triple-nudge-fatigue failure REQ RENEW-06 forbids.
- **Fix:** Dedup on `created_at` so the banner suppresses the instant the email is queued.
- **Files modified:** `src/lib/renewal/shouldShow.ts`
- **Verification:** Test 7 passes with a row that has no `sent_at` set at all.
- **Committed in:** `acfe909`
