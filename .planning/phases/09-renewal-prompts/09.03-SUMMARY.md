## Decisions Made

- **Uniform post-upsert pattern:** Rather than merging `original_gateway` into each existing upsert payload, every file uses a separate `.update(...).is("original_gateway", null)` call. This was mandated by the plan because `{onConflict:"id"}` upsert semantics replace the row — embedding the column would overwrite an existing non-null value on every subsequent webhook, breaking first-gateway-wins.
- **Best-effort writes:** The try/catch + warn-only design means a DB hiccup on this tag can never escalate into a payment provider retry storm. Payment ingestion stays sacred; renewal attribution is convenience.
- **Scope discipline:** Only the 4 files listed in plan frontmatter were touched. No refactors, no imports added, no response shape changes (CLAUDE.md S1–S3, rules 6–8).
