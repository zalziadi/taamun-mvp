## Founder Manual Safeguards (Pitfall #28 — plan-checker gap #1 fix)

These are **out-of-code** settings that must be configured once in the PostHog dashboard. They form a second defensive layer — if a code-side privacy setting ever regresses, these dashboard-level safeguards catch the leak.

| # | Safeguard | Expected value | Rationale |
|---|-----------|----------------|-----------|
| 1 | Usage alerts (Billing → Usage Alerts) | 70% + 90% of 1M/month free-tier quota | Expected load ≈ 450K/month; early warning before overage |
| 2 | Session Recording (Project Settings → Recordings) | OFF | Second layer: prevent recording of reflection pages if code setting regresses |
| 3 | Heatmaps (Project Settings → Toolbar) | OFF | Same reason as #2 |
| 4 | Autocapture (Project Settings → Autocapture) | OFF | We emit explicitly via `emitEvent()` only; autocapture would ingest verse/reflection text |
| 5 | Retention (Data Management → Event Definitions) | 30 days for non-funnel events; default for `day_complete` + `cycle_start` | Minimize PII exposure window; keep funnel data |
| 6 | Team Access | Founder + primary developer only | Least-privilege for admin console |

Catalog includes fields for `تاريخ آخر مراجعة` (last-review date) and `الشخص المسؤول` (owner) so audit trail is explicit.
