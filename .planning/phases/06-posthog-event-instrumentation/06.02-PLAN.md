<objective>
Add a runtime property whitelist guard inside `emitEvent()` so that even a future developer who bypasses the TypedEvent union (via `as any`, or via a new event added without review) cannot leak PII to PostHog. The CI grep rule (Plan 06.06) is the BUILD-TIME defense; this plan is the RUNTIME defense. Two gates, one banned list.

Purpose: ANALYTICS-12 demands that banned property-name patterns (`*_email`, `*_phone`, `reflection_*`, `verse_*`, `journal_*`, `message_*`, `prayer_*`) never reach PostHog. A TypeScript-only guard is bypassable. A runtime guard is not.

- `src/lib/analytics/server.ts` modified so `emitEvent()` calls `assertAllowedProperties(event.properties)` BEFORE the fetch

    ```typescript
    // Banned property-name patterns per ANALYTICS-12.
    // Source of truth: 06-CONTEXT.md §"Property whitelist".
    // CI grep (Plan 06.06) is the build-time defense; this is the runtime defense.
    export const BANNED_PROPERTY_PATTERNS: readonly RegExp[] = [
      /_email$/,        // *_email
      /_phone$/,        // *_phone
      /^reflection_/,   // reflection_*
      /^verse_/,        // verse_*
      /^journal_/,      // journal_*
      /^message_/,      // message_*
      /^prayer_/,       // prayer_*
    ] as const;

    export function assertAllowedProperties(props: Record<string, unknown>): void {
      for (const key of Object.keys(props)) {
        for (const pattern of BANNED_PROPERTY_PATTERNS) {
          if (pattern.test(key)) {
            throw new Error(
              `[analytics] Property "${key}" matches banned pattern ${pattern}. ` +
              `PII is never emitted to PostHog. See ANALYTICS-12 / 06-CONTEXT.md.`
            );
          }
        }
      }
    }
    ```
