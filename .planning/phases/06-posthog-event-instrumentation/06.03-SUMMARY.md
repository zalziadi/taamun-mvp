# Phase 6 Plan 03: PageviewTracker + Excluded Paths Summary

**One-liner:** Wired Suspense-wrapped App Router pageview tracking with a 6-prefix sacred-path exclusion list; the ONLY client-side PostHog call site in the entire v1.2 codebase.

---

## What Was Built

### 1. `src/lib/analytics/excludedPaths.ts`

A 2-export module that is the single source of truth for sacred-path exclusions:

- `EXCLUDED_PATH_PREFIXES: readonly string[]` — frozen array of `/day`, `/reflection`, `/book`, `/program/day`, `/api/guide`, `/guide`. Readonly typing allows Plan 06.06's CI grep lint to import and iterate without mutation risk.
- `isExcludedPath(pathname: string): boolean` — matches exact prefix OR subpath (prefix + `/`). So `/day` → true, `/day/7` → true, `/dayhint` → false (no trailing-slash match), `/program` → false (only `/program/day/*` is sacred).

### 2. `src/lib/analytics/excludedPaths.test.ts`

13 vitest cases covering the full behavior contract from the plan:

- 8 excluded-path assertions (`/day`, `/day/7`, `/reflection/abc`, `/book`, `/book/chapter-1`, `/program/day/5`, `/api/guide/ask`, `/guide/session/1`)
- 4 non-excluded-path assertions (`/`, `/pricing`, `/account`, `/program`)
- 1 export-shape assertion (confirms CI grep can consume `EXCLUDED_PATH_PREFIXES`)

All pass. RED→GREEN cycle followed strictly (test committed before implementation).

### 3. `src/components/PageviewTracker.tsx`

A ~40 LOC client component using `usePathname()` + `useSearchParams()` from `next/navigation`. On pathname/search change:

1. If pathname empty → no-op
2. If `isExcludedPath(pathname)` → no-op (ANALYTICS-09 / PROJECT.md principle #4)
3. If `!posthog.__loaded` → no-op (graceful degradation when env vars missing)
4. Otherwise: `posthog.capture("$pageview", { $current_url: url })` with the optional search string attached.

Returns `null` — no DOM output.

### 4. `src/components/AnalyticsProvider.tsx` (edited)

Preserved the existing `initAnalytics()` useEffect (CLAUDE.md safety rule S5). Added a `<Suspense fallback={null}>` boundary wrapping `<PageviewTracker />`. Without this boundary `useSearchParams()` opts the whole subtree into CSR bailout per React 18 / Next.js 14.

## Decisions Made

1. **Direct `posthog.capture` over `track()` wrapper.** The existing `track()` in `src/lib/analytics.ts` auto-merges stored UTM data into every event. UTM is already attached to the PostHog session at `identify()` time in `initAnalytics()` — re-attaching per pageview would double the network payload on every route change. `$pageview` goes direct.

2. **`posthog.__loaded` guard, NOT a new `isAnalyticsInitialized()` export.** Verified in `node_modules/posthog-js/dist/module.d.ts:2598` — `__loaded: boolean` is a public field on the PostHog class. Using it avoids editing `src/lib/analytics.ts`, which Plan 06.01 is also modifying in parallel (file-overlap avoidance per parallel_execution directive in the spawn prompt).

3. **`EXCLUDED_PATH_PREFIXES` readonly-array export shape.** Chose plain `readonly string[]` over `Set<string>` or object map so Plan 06.06's CI grep (which will likely be a Node script, not a TS consumer) can read and iterate the constant via simple dynamic import.

4. **`/program` deliberately NOT excluded.** Only `/program/day/*` is sacred. The dashboard-style `/program` route is telemetry-appropriate — it shows overall progress, renewal banners, etc. Confirmed against plan's behavior list: `isExcludedPath("/program")` must return `false`.
