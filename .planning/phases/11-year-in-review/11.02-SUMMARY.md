## Decisions Made

- **Disjoint by naming, not intersection.** `YIRPublicStats.reflections_count` (number, public) vs `YIRPrivateContent.reflection_text` (string, private). No overlap — `Extract<keyof A, keyof B>` is `never`.
- **Nullable timestamps + awareness_avg.** `earliest_reflection_at` / `latest_reflection_at` / `awareness_avg` all accept `null` to handle cold-start users (< 30 days) whose aggregation legitimately has no data yet.
- **Narrow guard, not zod.** `isYIRPublicStats` uses hand-rolled typeof/Array.isArray checks — satisfies NFR-08 ("no new deps") while still giving Plan 11.03 a runtime rejection point.
- **Exactly 1 grep match for `YIRPrivateContent` in types.ts.** Per the plan's verification clause, JSDoc references to the private type were replaced with descriptive prose ("private body" / "user-authored content") so the identifier appears only at its interface declaration — keeps Plan 11.07's grep guard scope surgical.
