## Accomplishments

- New route `src/app/year-in-review/og/route.tsx` (197 lines) — mirrors the Phase 10 referral OG structure verbatim with year-review-specific validation (YEAR_KEY_PATTERN instead of FRIEND_CODE_REGEX) and analytics (year_review_shared instead of none).
- Edge runtime declaration (`export const runtime = "edge"`) confirmed by `next build` route-manifest output (`ƒ /year-in-review/og` — dynamic/server-rendered).
- Query-param-only design: zero imports from `supabaseAdmin`, `getYearInReview`, `requireUser`, `cookies()`, `@/lib/reflections/*`, or `@/lib/awareness/*`. The endpoint is a pure function of `year_key` + `c` query params, which originate from YIRPublicStats values by construction (Plan 11.05 builds the URL).
- Regex gate: invalid `year_key` returns `400` plaintext before any image-render work — prevents the endpoint from being abused as an arbitrary text-to-image renderer (PITFALL #22 carried over from Phase 10).
- Count clamp: `c` is parsed, NaN-guarded, and clamped to `[0, 9999]` so large/adversarial integers cannot inflate digit strings in the rendered image.
- Latin numerals via `Intl.NumberFormat("en-US")` per YIR-05 cross-platform readability decision. Eastern Arabic glyph support on WhatsApp / Instagram crawlers is inconsistent; Latin keeps the preview stable.
- Fire-and-forget `year_review_shared` PostHog event emitted server-side on successful image response. `distinct_id` is the literal `"share_card_crawler"` — crawlers are not users, so no user identifier is ever attached. Failures are swallowed by a try/catch so analytics can never break the image response.
- Inlined PostHog capture fetch (not importing `@/lib/analytics/server`, which is Node-flavored and incompatible with the edge bundler). Properties `year_key` + `reflections_count` are both on the `ALLOWED_PROPERTY_KEYS` whitelist defined in `src/lib/analytics/events.ts`.
- Zero mentions of the private-body type identifier anywhere in the file (including comments) — Plan 11.07's grep guard scope stays surgical.

## Decisions Made

- **Query-param-only over edge-DB-read.** Edge runtime cannot bundle `supabaseAdmin` (Node `crypto` dep), and RLS blocks the anon key from reading `year_reviews`. Query-param input from Plan 11.05's URL builder is both the only feasible design AND a privacy win — the endpoint literally cannot reach personalization.
- **Latin numerals only on card.** Eastern Arabic glyphs are inconsistent on crawler renderers; Intl.NumberFormat("en-US") guarantees 0–9 glyphs. Matches Phase 10 OG precedent (Latin-only body for v1.2).
- **`distinct_id = "share_card_crawler"` literal.** Crawlers fetching the image are not users. Using the user's anonymous hash or email would either be wrong (not the user) or a PII leak. The literal string measures crawler-render reach; upstream click-time share-intent tracking lives in Plan 11.05.
- **Inlined PostHog fetch instead of `emitEvent`.** `@/lib/analytics/server` uses Node-flavored guards/imports that edge bundler rejects. The capture payload is small and stable enough to hand-roll once.
- **Zero references to the private-body type identifier in this file.** Same Plan 11.02 technique: JSDoc and inline comments describe the split using prose ("private body type", "user-authored content") instead of naming the identifier. Keeps Plan 11.07's grep guard scope surgical.
- **Count clamp instead of error.** An out-of-range `c` value silently clamps to `[0, 9999]`. Crawlers that get a 400 don't preview; silent clamp keeps the share preview working even for malformed inputs (while YEAR_KEY_PATTERN still 400s for the abuse-vector case).
