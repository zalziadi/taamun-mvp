## Decisions Made

- **Shape-mirror over a custom Node ESM loader.** Node ESM can't resolve `./types` (without `.ts`) nor `@/*` aliases without a custom loader, and NFR-08 forbids adding deps. Phase 09 harness precedent: shape-mirror the helper logic inline, then add drift detection (Scenario G greps the real source for the constants). Harness stays honest because (a) any drift in constants fails Scenario G, (b) the phase-11 guard greps the real source files, and (c) `guard:release` chains tsc + build + guards so the real code is exercised independently.
- **Scenario G added as drift-detector.** Plan required 6 scenarios; adding a 7th constant-sanity test costs ~20ms and protects against the inline mirrors going stale. Defense-in-depth precedent from Phase 10 harness (scenario 7 privacy sweep).
- **Regression-insurance self-test behind a flag.** `--no-regression-test` skips the file-mutation self-test for CI environments. Default (local dev) runs it so every manual invocation proves the grep actually catches violations.
- **Comment-line carve-out in the guard.** Same pattern as Phase 09/10 — lines whose content (after `path:linenum:`) starts with `*` or `//` are skipped. JSDoc blocks documenting bans (e.g. "⚠ never call X") don't trip the guard, but the same string inside real JSX / imports / string literals still matches. Verified by injection test.
- **6 buckets, not the plan's suggested 3.** Plan's action block described 4 buckets (og, page, ui, lib) plus 2 file-level checks (package.json, required-title). This summary adopts 6 because the package.json check is independent of file scope and the required-title check is independent of ban scope. Same logical coverage, cleaner output.
- **Pre-existing phase-07 guard failure logged, not fixed.** Scope boundary rule: only auto-fix issues DIRECTLY caused by the current task's changes. Phase 07 guard's failure on MilestoneBadge.tsx:16 JSDoc predates Plan 11.07 (confirmed by `git stash` + re-run). Logged in `deferred-items.md`.

## Issues Encountered

- **Node ESM + TypeScript-strip limitations**: the experimental `--experimental-strip-types` flag DOES strip `.ts` extensions, but transitive imports like `yearKey.ts` → `./types` (without `.ts`) still fail because Node ESM's resolver requires extensions. Workaround: shape-mirror inline + drift-detection (Scenario G). Phase 09 precedent.
- **Carve-out regex edge case**: initially concerned that the injected-comment test would appear to succeed because of the carve-out. Verified via direct regex test that the carve-out correctly skips `//`-prefixed lines, and then verified that real-code injection (`const leak = "YIRPrivateContent"`) DOES trigger the guard. Both behaviors correct.
