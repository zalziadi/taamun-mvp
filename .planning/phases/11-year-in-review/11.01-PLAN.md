    **Section 1: year_reviews table** (per YIR-03)
    ```sql
    create table if not exists public.year_reviews (
      id           uuid         primary key default gen_random_uuid(),
      user_id      uuid         not null references auth.users(id) on delete cascade,
      year_key     text         not null,                 -- e.g., '2027_anniversary'
      payload      jsonb        not null,                  -- YIRPublicStats shape (see Plan 11.02)
      generated_at timestamptz  not null default now(),
      constraint year_reviews_unique_pair unique (user_id, year_key)
    );
    ```

    **Section 2: Indexes**
    ```sql
    create index if not exists idx_year_reviews_user on public.year_reviews (user_id);
    -- Required for fast RPC range aggregation (YIR-12). Composite index, not separate.
    create index if not exists idx_reflections_user_created on public.reflections (user_id, created_at);
    ```
    **DO NOT** add indexes on awareness_logs / badges / user_progress — grep for existing `user_id` indexes on those tables first and only add what's missing. The CONTEXT says reflections + awareness_logs indexes already exist; awareness_logs likely does (verify and comment "already indexed" if so).

    **Section 3: RLS** (per YIR-01, REFER-12 pattern)
    ```sql
    alter table public.year_reviews enable row level security;
    do $$ begin
      if not exists (select 1 from pg_policies where policyname = 'year_reviews_select_own') then
        create policy year_reviews_select_own on public.year_reviews
          for select using (auth.uid() = user_id);
      end if;
    end $$;
    -- NO insert/update/delete policies — service role only (upsert lives in src/lib/yearInReview/aggregate.ts, Plan 11.03)
    ```

    **Section 4: RPC `get_year_in_review(p_user_id uuid, p_year_key text) returns jsonb`** (per YIR-02)

    The RPC resolves the anniversary window from `profiles.activation_started_at` (fallback `profiles.created_at`) based on the `year_key` input (format `YYYY_anniversary`). Inside the function:
    - `window_start := activation_anniversary_for_year(p_year_key)`; `window_end := window_start + interval '1 year'`
    - Use `security definer` with `set search_path = public, auth` so it respects RLS for ownership checks but can read aggregates cleanly
    - Build result via `jsonb_build_object(...)` with subqueries:
      - `reflections_count` := `count(*) from reflections where user_id=p_user_id and created_at >= window_start and created_at < window_end`
      - `awareness_avg` := `avg(value) from awareness_logs where ...` (null-safe via `coalesce` or leave null)
      - `milestones_reached` := `coalesce(json_agg(distinct badge_code order by badge_code), '[]'::json)::jsonb from badges where user_id=p_user_id and unlocked_at between window_start and window_end`
      - `cycle_count` := `coalesce(max(current_cycle), 1) from user_progress where user_id=p_user_id`
      - `earliest_reflection_at`, `latest_reflection_at` := `min()` / `max()` of `reflections.created_at` in range
      - `awareness_trajectory` := bucketed averages per week: `array_agg(round(avg(value)*100)/100 order by week_num) from (select date_trunc('week', created_at) as week_num, value from awareness_logs where ... ) s group by week_num` — wrapped so result is a jsonb array of numbers (≤52 elements)
    - Function MUST NOT select any reflection text / emotion labels / prayer content — only aggregates. Comment this constraint explicitly in the function header (privacy guarantee at the data-layer — Plan 11.06 is the type-layer guarantee).
    - `grant execute on function public.get_year_in_review(uuid, text) to authenticated;`

    **VERIFY COLUMN NAMES FIRST** before writing the RPC — read `supabase/migrations/20260309000000_daily_journey_tables.sql` (reflections), `20260419000000_v1_2_badges_and_cycle_guard.sql` (badges — confirm `unlocked_at`, `badge_code`), and the migration that created `user_progress` (confirm `current_cycle`, not `cycle_number`). If a column name differs from what CONTEXT suggests, follow the actual schema and note it in a `-- NOTE:` comment in the migration.

    **Section 5: Commented DOWN block** — copy the structure from `20260422000000_v1_2_referrals.sql`. Never execute; operator reference only.

    Banned (anti-patterns per CONTEXT §R4 + NFR-09):

    - No `as any`-equivalents in the RPC (untyped `json_agg` without cast is fine; but always `::jsonb`)
    - Do NOT name the table `yir_snapshots` or `year_in_review_cache` — must be `year_reviews` exactly (CONTEXT + YIR-03)
