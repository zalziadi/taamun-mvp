## Decisions Made

- **Anchor fallback:** `profiles.activation_started_at` is not in the schema today. The RPC uses `profiles.created_at` directly (equivalent to `COALESCE(activation_started_at, created_at)` — plan CONTEXT pre-approved this fallback). Documented inline.
- **Awareness scalar mapping:** Plan assumed `awareness_logs.value` (numeric). The actual column is `level text` with values `'present'|'tried'|'distracted'`. Implemented a CASE expression mapping to `1.0 / 0.5 / 0.0` for both `awareness_avg` and the weekly-bucketed `awareness_trajectory`. Documented in the schema-realities block at the top of the migration.
- **Cycle count:** Derived as `GREATEST(current_cycle, array_length(completed_cycles, 1))` from `public.progress` (plan said `max(current_cycle)` — more accurate given the `completed_cycles int[]` column added in `20260418000000_add_cycles_support.sql`).
- **Malformed year_key guard:** Returns empty aggregates JSON rather than raising; keeps `/year-in-review` rendering a friendly empty state if a bad key ever reaches the RPC.
