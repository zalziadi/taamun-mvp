## Decisions Made

- **oxc over esbuild in vitest config.** First attempt set `esbuild.jsx = "automatic"`. Vitest 4 responded with `Both esbuild and oxc options were set. oxc options will be used and esbuild options will be ignored.` — their primary transform is oxc (via rolldown). Switched to `oxc.jsx.runtime = "automatic"` which is the supported shape.
- **.test.ts (not .test.tsx) for the test file.** Project has zero `.test.tsx` files — all 48 existing test files are `.ts`. To stay consistent, the new test uses `createElement(AwarenessTrajectory, props)` + `renderToStaticMarkup(...)`. The component file itself (`.tsx`) still gets JSX-transformed when imported.
- **`@` alias in vitest config.** Component imports `@/lib/yearInReview/types`. Vitest needs the alias explicitly since tsconfig paths aren't auto-loaded. `resolve({ alias: { "@": resolve(__dirname, "./src") } })`.
- **Source-file grep as a test invariant.** Test #7 does `readFileSync(AwarenessTrajectory.tsx)` and asserts zero chart-library identifiers. This survives future refactors in a way that a one-time CLI check cannot.
- **Privacy comments rewritten without the identifier.** Original draft had "never touches `YIRPrivateContent`" in the component JSDoc; rewritten to "never any user-authored content (reflection text / emotion labels / guide messages)". Keeps Plan 11.07's grep guard surgical regardless of whether it's word-based or import-based.
- **Tone comments avoid banned vocabulary even in negations.** Original draft said `NO "ranked", NO "top X%", NO "unlocked"`; rewritten to `no ranking vocabulary, no percentile comparisons, no gamified milestone vocabulary`. Defense-in-depth against any future word-level guard.
- **cycle_count conditional render.** Displayed only when `> 0`. Avoids showing "أكملت ٠ حلقة" for early-retrospective users.
- **Sparkline and milestone sections both conditional on non-empty data.** `length >= 2` gate on trajectory (single-point can't draw a line) and `length > 0` gate on milestones_reached. Cleaner UI for partial-year users.

## Deviations from Plan

- **[Rule 3 – Blocking issue] Test file written as `.test.ts` (not `.test.tsx`) and a new `vitest.config.ts` added.** The plan's action block showed a `.test.tsx` file importing JSX directly. First run failed: vitest 4's default transform couldn't parse JSX (project `tsconfig.json` sets `jsx:"preserve"`). Two paths forward: (a) add vitest config with a JSX loader, OR (b) rewrite test to avoid JSX in the test file. Applied both — added minimal `vitest.config.ts` for the component's own JSX transform, AND kept test file as `.ts` using `createElement(...)` for convention parity with the 48 existing `.ts` test files. Rationale: project had zero `.test.tsx` files before, so introducing one is a structural shift; the hybrid approach achieves all 7 invariants the plan required without that shift.
- **[Rule 2 – Auto-add missing critical functionality] `cycle_count` wrapped in `> 0` conditional.** Plan's action block rendered `{num(stats.cycle_count)}` unconditionally, which would print "٠ حلقة" for early-retrospective users — awkward display. Added the conditional; identical behavior for users with any completed cycle.
- **[Rule 1 – Bug] `YIRPrivateContent` and banned-vocab words removed from comments.** Plan 11.07's grep guard scope isn't final, but comments containing even negated occurrences of `YIRPrivateContent` / `unlocked` / `ranked` / `top X%` could trip word-based guards. Rewrote both comment blocks to use prose equivalents. Zero behavioral change.

- **`esbuild.jsx` ignored warning.** Vitest 4 uses oxc as primary transform. Config migrated to `oxc.jsx.runtime="automatic"`. No warning now.
