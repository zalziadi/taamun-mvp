# Phase 08 Plan 01: Widen MilestoneBadge — Summary

Widened `MilestoneBadge.tsx` from a single-variant (day_28) presentational SVG into a 7-variant discriminated-union renderer — zero new deps, zero hooks, zero schema change — unblocking Plans 08.02/08.03/08.05.

## What was built

### Widened BadgeCode union (exported)

```typescript
export type BadgeCode =
  | "day_1"
  | "day_3"
  | "day_7"
  | "day_14"
  | "day_21"
  | "day_28"
  | "cycle_complete";
```

Previously `type BadgeCode = "day_28"` (non-exported). Downstream plans now import this type.

### Final Arabic copy (classical register, no English-loan vocabulary)

| code             | name                     | numeral |
| ---------------- | ------------------------ | ------- |
| `day_1`          | عتبة البداية              | ١        |
| `day_3`          | عتبة الثلاث              | ٣        |
| `day_7`          | عتبة السبع               | ٧        |
| `day_14`         | عتبة الأربعة عشر         | ١٤       |
| `day_21`         | عتبة الواحد والعشرين     | ٢١       |
| `day_28`         | عتبة الثامن والعشرين     | ٢٨       |
| `cycle_complete` | تمام الحلقة              | ◯        |

Eastern-Arabic numerals render only as HTML `<span>` siblings — never inside SVG `<text>`.

### Final SVG geometry per variant

All variants share: `viewBox="0 0 120 120"`, width/height 96, stroke `#C9A24B`, fill `none`, no gradient, no glow, no animation.

| code             | geometry                                                                                   |
| ---------------- | ------------------------------------------------------------------------------------------ |
| `day_1`          | single centered circle r=10 + one top cardinal tick                                        |
| `day_3`          | outer circle r=54 + inscribed equilateral triangle + inner circle r=14                     |
| `day_7`          | circle r=50 + 7 radial ticks spaced 360/7 deg from r=50 to r=58                            |
| `day_14`         | concentric r=54/36 + 14 small dots (r=1.2) on a r=44 arc, spaced 360/14 deg                |
| `day_21`         | 3 concentric r=54/40/14 + 7 radial ticks from r=54 to r=60 (3 × 7 = 21 visual cue)         |
| `day_28`         | PRESERVED from Phase 7 — concentric r=54/42/10 + 4 cardinal ticks                          |
| `cycle_complete` | solid outer ring r=54 strokeWidth=2 + inner day_28 geometry (concentric + cardinal ticks)  |

### Structural invariants preserved

- Wrapper (`<div dir="rtl">` + `inline-flex flex-col items-center gap-2` + `opacity-40 grayscale` when locked) unchanged from Phase 7
- Arabic name + numeral remain HTML `<span>` siblings (PITFALLS.md #8)
- No `"use client"` — component stays presentational, parent decides client/server boundary
- No hooks (`useState`/`useEffect`), no `fetch()`, no layer leak
- No new imports (no framer-motion, no lottie, no canvas-confetti, no next/og)
