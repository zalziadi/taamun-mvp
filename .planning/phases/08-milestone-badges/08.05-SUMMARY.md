# Phase 08 Plan 05: Badge Grid on `/progress` — Summary

**One-liner:** Added a private `BadgeGrid` client component to `/progress` that renders 7 milestone badges per cycle (up to 3), subtle opacity fade-in on unlocked ones, dim on locked — no modal on click, no share button, no next/og route, no "Unlocked!" copy, no particles.

## What was built

### `src/components/badges/BadgeGrid.tsx` — 214 lines, new

- `"use client"` component; imports `MilestoneBadge` + `BadgeCode` from Plan 08.01's widened component.
- On mount: `createBrowserClient(...).from("badges").select("badge_code, cycle_number, unlocked_at").eq("user_id", <auth.user.id>).order("cycle_number")`.
  - RLS policy `users_select_own_badges` (Phase 7 migration) scopes rows to the authenticated user; no cross-user leakage possible.
  - Query returns ≤ 21 rows per user (7 codes × 3 cycles cap) — single indexed SELECT on `idx_badges_user_id`.
- Render structure:
  - `<section dir="rtl" aria-label="شارات الرحلة">`
  - Header row: "شارات الرحلة" + private-copy chip "خاصة بك — لا تُشارَك"
  - For each cycle `1..maxCycle` (capped at 3):
    - Subheader "الحلقة ١/٢/٣" (Eastern-Arabic numerals)
    - `grid-cols-4 sm:grid-cols-7` of 7 `MilestoneBadge` slots
    - Per slot: `<motion.div initial={unlocked ? {opacity:0} : false} animate={{opacity: unlocked ? 1 : 0.4}} transition={{duration:0.6, ease:"easeOut"}} title={tooltip}>`
- Loading state: 7 animated skeleton circles (aria-hidden).
- Error/empty state: `return null` (never interrupts the page).

### `src/app/progress/page.tsx` — +4 lines, modified

- Added one import: `import { BadgeGrid } from "@/components/badges/BadgeGrid";` (line 24).
- Added one render site: `{status === "ok" && <BadgeGrid />}` (line 283, between the Bridge section and the Phase sections).
- No other changes — existing page structure byte-identical elsewhere (S3 minimum diff honored).

### Behavioural confirmations (plan-level)

- Guest user (no Supabase session) → `BadgeGrid` short-circuits at `authData.user == null` → `setStatus("empty")` → returns `null` → grid not rendered. Plus the page-level `status === "ok"` gate prevents even the empty-status flicker for offline/unauthenticated states.
- Click on a badge → no handler attached → browser's default passthrough → nothing opens. No `<button>`, `<Link>`, `onClick`, `onMouseDown`, or any interactive wrapper.
- Earned badges (from Plan 08.02 live unlocks + Plan 08.04 backfill) render at opacity 1 (plus MilestoneBadge's default stroke). Unearned render at opacity 0.4 combined with MilestoneBadge's internal `opacity-40 grayscale` dim — producing the double-layer dim effect the plan specifies.
- RTL order: `dir="rtl"` on the section + default grid flow → first badge (day_1) sits on the visual right, reads right-to-left across the row on both mobile (4-col) and desktop (7-col).
- Cycle cap: `Math.min(3, Math.max(1, max))` → user with `completed_cycles = [1, 2, 3, 4]` still only sees 3 cycle rows.
