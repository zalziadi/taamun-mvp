## Deviations from Plan

### Rule 1 — Guard regex false positive on docblock

**Found during:** Task 2 first-run verification of `guard:phase-08`.

**Issue:** Initial vocabulary ban regex (`/"[^"]*(Unlocked!|Achievement|...)[^"]*"/`) matched line 16 of `src/components/badges/MilestoneBadge.tsx`:

```js
// * - No English-loan vocabulary for milestones ("Unlocked!", "Achievement").
```

This is a JSDoc continuation line legitimately documenting the component's tone ban — exactly the kind of self-documentation 08.01's SUMMARY.md `decisions` field preserves. Forcing a rewrite of 08.01's docblock would violate the cross-plan scope boundary.

**Fix:** Narrowed the guard's vocab check to pipe through `grep -v ':[[:space:]]*\*|:[[:space:]]*//'`, skipping lines whose first non-whitespace character is `*` or `//`. Documented the carve-out in the guard's own inline comment with the precedent reference (07.04 Rule 1 + 08.05's affirmative-docstring rewrite). Re-verified that a real JSX violation (`alert("Unlocked!")`) still trips the guard as intended.

**Files modified:** `scripts/guards/phase-08-anti-patterns.sh` (inline before first commit).

**Commit:** `c4b642b` — fixed before committing.
