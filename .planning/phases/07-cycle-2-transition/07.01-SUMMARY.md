# Phase 7 Plan 01: Badges Table + Cycle-Paused-At Column Summary

**One-liner:** Additive Supabase migration creates `public.badges` with the composite UNIQUE constraint that Plan 07.04's idempotent Day-28 unlock depends on, and adds the nullable `progress.cycle_paused_at` column Plan 07.02's optimistic-concurrency guard reads.

## What shipped

A single SQL file at `supabase/migrations/20260419000000_v1_2_badges_and_cycle_guard.sql` that:

1. **Adds** `progress.cycle_paused_at timestamptz` via `ALTER TABLE ... ADD COLUMN IF NOT EXISTS`. Nullable, no default, no NOT NULL — strictly additive per NFR-09.
2. **Creates** `public.badges` table with columns: `id uuid PK`, `user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`, `badge_code text NOT NULL`, `cycle_number int NOT NULL CHECK (1..99)`, `unlocked_at timestamptz NOT NULL DEFAULT now()`, `notified boolean NOT NULL DEFAULT false`.
3. **Enforces** `UNIQUE (user_id, badge_code, cycle_number)` — load-bearing for Plan 07.04's `ON CONFLICT DO NOTHING` idempotent insert.
4. **Indexes** `idx_badges_user_id` on `user_id` for `/progress` page reads.
5. **Enables** RLS and creates a single `users_select_own_badges` policy (SELECT-only, authenticated role, `auth.uid() = user_id`). No INSERT/UPDATE/DELETE policies — writes go through `getSupabaseAdmin()` service role only.
6. **Documents** a manual DOWN block in a trailing comment so a human operator can revert in an incident.

### The CREATE TABLE statement actually used

```sql
CREATE TABLE IF NOT EXISTS public.badges (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  badge_code text NOT NULL,
  cycle_number int NOT NULL CHECK (cycle_number >= 1 AND cycle_number <= 99),
  unlocked_at timestamptz NOT NULL DEFAULT now(),
  notified boolean NOT NULL DEFAULT false,
  UNIQUE (user_id, badge_code, cycle_number)
);
```
