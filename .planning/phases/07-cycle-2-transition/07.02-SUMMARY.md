# Phase 7 Plan 02: start-cycle Race Guard + cycle_start Emission Summary

**Race-guarded `/api/program/start-cycle` with 409 cycle_race response + single `emitEvent("cycle_start")` call site + narrowed `42703`-only fallback + first write of `progress.cycle_paused_at` — delivers ROADMAP Phase 7 Success Criterion #3 (multi-device race test) at the server boundary.**

## Accomplishments

- Route handler now accepts `expected_current_cycle` in the POST body and returns `409 { error: "cycle_race", current_cycle }` when the DB state disagrees — no mutation, no analytics event.
- Two-step concurrency guard: first SELECT compares `dbCurrentCycle` vs `expectedCurrentCycle`; then UPDATE is guarded by `.eq("user_id", userId).eq("current_cycle", expected)` and inspects `data.length === 0` to catch the DB-level race window between SELECT and UPDATE.
- The same `.update({...})` call ALSO writes `cycle_paused_at: new Date().toISOString()` — closes plan-checker gap #1 from Plan 07.01 by making the column non-dead.
- Silent fallback narrowed: the legacy "retry without cycle columns" block now runs ONLY when `upsertErr.code === "42703"` (PostgreSQL `undefined_column`). Any other error class returns 500 immediately.
- `emitEvent("cycle_start", ...)` fires exactly once on the 2xx path with typed properties `{ new_cycle_number, prior_cycle_days_completed, tier }`. Never on 401 / 409 / 500.
- `tier` sourced from `profiles.subscription_tier` (consistent with `src/app/api/program/progress/route.ts`) with graceful fallback to `"unknown"`.

## Unified-diff highlights

- `+ import { emitEvent } from "@/lib/analytics/server";`
- `+ const body = (await req.json().catch(...)) as { cycle?: number; expected_current_cycle?: number };`
- `+ if (dbCurrentCycle !== undefined && dbCurrentCycle !== expectedCurrentCycle) { return NextResponse.json({ error: "cycle_race", current_cycle: dbCurrentCycle }, { status: 409 }); }`
- `+ .update({ ..., cycle_paused_at: new Date().toISOString(), ... }).eq("user_id", userId).eq("current_cycle", expectedCurrentCycle).select();`
- `+ if (!updated || updated.length === 0) { return NextResponse.json({ error: "cycle_race", ... }, { status: 409 }); }`
- `+ // PITFALL #2: silent fallback narrowed to undefined_column only — do not swallow other errors.`
- `+ if (upsertErr) { if (upsertErr.code === "42703") { /* minimal retry */ } else { return 500; } }`
- `+ void emitEvent({ name: "cycle_start", properties: { new_cycle_number, prior_cycle_days_completed, tier } }, userId);`

Previous behavior removed: the unconditional `.upsert(fullPayload, { onConflict: "user_id" })` path and the unconditional `upsertErr` fallback block.
