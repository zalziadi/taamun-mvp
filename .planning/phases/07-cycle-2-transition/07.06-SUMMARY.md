## Deviations from Plan

### Auto-fixed issues

**1. [Rule 1 — Bug] Shared-global PostHog attribution fails under Promise.all**
- **Found during:** Task 2 first run — Scenario 2's "race loser contributed 0 events" assertion failed 2 !== 0
- **Issue:** The harness initially used a shared mutable `currentInvocation` global that both concurrent Promise.all callers overwrote. Events ended up tagged with whichever value was set last.
- **Fix:** Plumbed `invocation` through the call chain explicitly (emitEvent → unlockBadge → startCyclePOST signatures), and attributed captures via an `X-Invocation` HTTP header inside the fetch interceptor.
- **Files modified:** `scripts/verify/phase-07-integration.mjs` only
- **Commit:** folded into `81674b0`

**2. [Rule 3 — Scope] Guard's `streak` ban would trip 40+ legitimate library files**
- **Found during:** Task 3 planning pre-check
- **Issue:** The plan listed `streak` as a banned vocab term, but `streak` is a heavily-used progress metric across `src/lib/progressEngine.ts`, `src/lib/personalityEngine.ts`, `src/lib/streak.ts`, `src/lib/engagement/anticipation.ts`, etc. Blanket grep would produce 40+ false positives and make the guard unusable.
- **Fix:** The guard's vocab check is scoped to sacred components + sacred routes and matches only QUOTED forms (`"Unlocked!"`, `"streak!"`, `"🔥 N days streak"`) — the user-facing shape, not the structural metric name. Library code using `streak: number` as a prop type is unaffected.
- **Files modified:** scope decision documented in the guard's header comment.
- **Commit:** folded into `c4d64c9`

**3. [Rule 3 — Scope] Harness cannot import route handlers directly**
- **Found during:** Task 2 planning pre-check
- **Issue:** Node ESM cannot resolve `@/lib/*` path aliases without a custom loader. Next.js route handlers also require a request context (`cookies()` throws outside one). NFR-08 forbids adding a test-runner / loader / mocking dep.
- **Fix:** Harness imports the REAL `events.ts` (no aliases, works via relative path) for the property-whitelist guard, and re-implements the route handler + unlockBadge sequence 1:1 against an in-memory fake Supabase. The invariants being proven (exactly-once emission, idempotent insert, 409 silent loser) are sequence-level — they're correctly provable against the same-shape re-implementation. Documented at length in the harness header.
- **Files modified:** n/a — architectural decision
- **Commit:** folded into `81674b0`
