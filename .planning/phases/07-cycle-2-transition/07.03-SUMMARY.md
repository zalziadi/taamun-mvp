# Phase 7 Plan 3: Inline Day-28 "واصل الرحلة" CTA Summary

**One-liner:** Inline Arabic-native CTA inside DayExperience that, on Day 28 after reflection is saved, posts to the race-safe `/api/program/start-cycle` route — framer-motion fade only, no modal, no confetti.

## What shipped

An inline `<section>` wrapped in `<AnimatePresence>` + `<motion.section>` appended to the end of the day view. Rendered strictly when:

```ts
const isCompleted = progress?.completed_days?.includes(28) ?? false;
// ...
{day === 28 && isCompleted && ( ... )}
```

`progress` is fetched once on mount from `GET /api/program/progress` (the canonical client-facing endpoint — returns `completed_days` + `current_cycle`). No direct Supabase query from this component — the existing app convention is preserved.

## Click handler (race-safe)

```ts
const handleStartCycle2 = useCallback(async () => {
  if (ctaState !== "idle") return;           // double-tap guard
  setCtaState("submitting");                 // disables button immediately
  const currentCycle = progress?.current_cycle ?? 1;
  try {
    const res = await fetch("/api/program/start-cycle", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        cycle: currentCycle + 1,
        expected_current_cycle: currentCycle,  // RETURN-07 optimistic concurrency
      }),
    });
    if (res.ok) { router.refresh(); return; } // server redirects to day 1 / cycle 2
    if (res.status === 409) { setCtaState("raced"); return; } // other device won
    setCtaState("error");                      // 4xx/5xx — retry after 3s
    setTimeout(() => setCtaState(s => s === "error" ? "idle" : s), 3000);
  } catch {
    setCtaState("error");
    setTimeout(() => setCtaState(s => s === "error" ? "idle" : s), 3000);
  }
}, [ctaState, progress, router]);
```
