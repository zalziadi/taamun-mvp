Modify `src/app/api/program/start-cycle/route.ts` in-place. Apply the following changes without rewriting the file wholesale (respect CLAUDE.md S1-S3: read first, smallest diff possible).

1. **Extend body schema**: Accept an optional `expected_current_cycle?: number` alongside the existing `cycle?: number`. If omitted, default to `progress.current_cycle ?? 1` read from DB.

2. **Add optimistic-concurrency guard**: Replace the current unconditional `.upsert()` with a two-step pattern:

   a. Read current row: `const { data: current } = await admin.from("progress").select("current_cycle, completed_days, completed_cycles").eq("user_id", userId).maybeSingle();`

   b. Compute `expected = body.expected_current_cycle ?? current?.current_cycle ?? 1`.

   c. If `current?.current_cycle !== undefined && current.current_cycle !== expected`:
      return `NextResponse.json({ error: "cycle_race", current_cycle: current.current_cycle }, { status: 409 });` — DO NOT fire the analytics event.

   d. Attempt a conditional `.update()` (NOT upsert) when the row exists, guarded by `.eq("user_id", userId).eq("current_cycle", expected)`. Use `.select()` to get the row count. If `data.length === 0`, another writer won the race between our read and update — return 409.

   **IMPORTANT (plan-checker gap #1, 2026-04-19):** In the same `.update({...})` call, ALSO set `cycle_paused_at: new Date().toISOString()` — this earns the column its place in Plan 07.01's migration (bookkeeping of the transition moment; future race-detection windows can read it). Without this write, `progress.cycle_paused_at` would be dead code.

   e. If no row exists yet (new user edge case), fall back to `.insert()` with `user_id, current_day=1, current_cycle=targetCycle, completed_days=[], completed_cycles=[finishedCycle]`.

3. **Narrow the fallback path**: Per PITFALLS.md #2, the existing "retry without cycle columns" block currently runs on ANY error. Change it to run ONLY when `upsertErr.code === '42703'` (PostgreSQL's undefined_column error). For any other error code, return 500 immediately. Add a code comment: `// PITFALL #2: silent fallback narrowed to undefined_column only — do not swallow other errors`.

4. **Emit `cycle_start` event** on successful 2xx path only, BEFORE the NextResponse return. Import `emitEvent` from `@/lib/analytics/server` and `TypedEvent` from `@/lib/analytics/events`. Build the event:

   ```typescript
   await emitEvent(
     {
       name: "cycle_start",
       properties: {
         new_cycle_number: targetCycle,
         prior_cycle_days_completed: currentCompleted.length,
         tier: /* read from profiles.subscription_tier — fall back to "unknown" if unavailable */,
       },
     },
     userId
   );
   ```

   The `tier` value comes from a SELECT on `profiles.subscription_tier` for `userId`. If the SELECT fails or returns null, use the literal string `"unknown"` (never throw). Do NOT block the response on the event emission — `emitEvent` is already fire-and-forget / never-rejects per its contract.
